# Owner and agent APIs

These routes are for management and authenticated control, not public discovery.

## Owner-scoped endpoint management

### `POST /agent/endpoints`

Purpose:

- create a new agent endpoint
- requires an initial x402-backed payment

### `PUT /agent/endpoints?slug=...`

Purpose:

- top up endpoint credits
- update selected management state depending on the tool/flow

### `GET /agent/endpoints?slug=...`

Purpose:

- fetch endpoint status and operational state

### `DELETE /agent/endpoints?slug=...`

Purpose:

- remove an endpoint permanently

These flows are already described in the Agent Management guide.

## Wallet-first auth

### `POST /agent/auth/challenge`
### `POST /agent/auth/verify`

Purpose:

- prove wallet ownership
- exchange a signed challenge for a short-lived session token

This auth model is used by wallet-first ERC-8004 and related agent flows.

## ERC-8004 prepare/finalize routes

### `POST /agent/erc8004/prepare`
### `POST /agent/erc8004/finalize`

Use these to prepare and finalize wallet-first agent registration.

### `POST /agent/erc8004/update/prepare`
### `POST /agent/erc8004/update/finalize`

Use these to update already-registered agents.

### `GET /agent/erc8004/mine`
### `GET /agent/erc8004/endpoints/mine`

Use these to list owned agents and bindable endpoints after wallet auth.

## MCP control plane

MCP endpoint:

```text
https://mcp.x402layer.cc/mcp
```

Purpose:

- owner-wide inventory lookup
- endpoint and product management
- payment-backed flows
- agent registry tooling

Primary auth:

- PATs
- legacy endpoint keys where still supported

## Best use cases

Use this surface when you need:

- authenticated owner operations
- wallet-authenticated agent lifecycle actions
- MCP automation
- endpoint management or top-up flows
