# Marketplace discovery

Use these routes for public discovery and listing lookup. They do not require account auth.

## Marketplace index

### `GET /api/marketplace`

Base URL:

```text
https://api.x402layer.cc/api/marketplace
```

Purpose:

- list endpoints, products, and agentic listings
- filter by chain, mode, category, search, and type
- power marketplace browsing and agentic discovery

Common query params:

- `type`
- `chain`
- `mode`
- `category`
- `search`
- `limit`
- `offset`

Example:

```bash
curl "https://api.x402layer.cc/api/marketplace?type=agentic&chain=base"
```

## Public endpoint details

### `GET /api/public/endpoints/:slug`

Purpose:

- fetch public payment metadata for an endpoint
- resolve direct vs credit mode
- show chain and wallet/payment options before hosted checkout

Example:

```bash
curl "https://studio.x402layer.cc/api/public/endpoints/your-slug"
```

## Public product lookup

### Hosted product route

Products are primarily discovered through marketplace listings and hosted pay pages:

```text
https://studio.x402layer.cc/pay/:id-or-slug
```

The hosted pay page then resolves the product and coordinates the x402 product flow against:

```text
https://api.x402layer.cc/p/:slug
```

## ERC-8004 public discovery

Relevant public routes include:

- `GET /agent/erc8004/registry`
- `GET /agent/erc8004/:agentId`
- marketplace agent listing surfaces documented in the Agentic Access guides

## Best use cases

Use the marketplace discovery surface when you need:

- public catalog search
- category or chain filters
- listing detail lookup
- endpoint metadata before payment
- agent discovery
