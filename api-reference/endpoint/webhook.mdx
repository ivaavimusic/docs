# Webhooks and receipts

This surface covers the server-to-server trust layer after payment.

## Webhooks

Primary dashboard API routes:

- `GET /api/webhooks`
- `POST /api/webhooks`
- `PATCH /api/webhooks/:id`
- `DELETE /api/webhooks/:id`

Purpose:

- list webhook endpoints
- create a new seller webhook
- toggle or rotate secret
- delete a webhook

Delivery event coverage includes:

- endpoint payments
- product payments
- agent endpoint create
- agent endpoint top-up

See the full operational guide at `/payments/webhooks`.

## Receipt tokens

Receipt tokens are signed JWTs returned after successful payment. They let servers verify:

- payer wallet
- amount
- currency
- tx hash
- source slug
- correlation metadata

Key verification assets:

- SDK docs: `/payments/sdk-receipts`
- JWKS: `https://api.x402layer.cc/.well-known/jwks.json`

## Typical trust chain

1. Buyer pays through x402.
2. Worker settles payment.
3. Worker returns a signed receipt token.
4. Worker optionally dispatches a signed webhook to the seller.
5. Seller verifies:
   - webhook HMAC signature
   - receipt JWT signature
   - critical fields like `tx_hash`, `source_slug`, and `amount`
